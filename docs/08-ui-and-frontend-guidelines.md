# UI 및 프론트엔드 가이드라인

## 기본 기술 기준

- 프론트엔드는 React, TypeScript, Vite를 기본으로 한다.
- 화면과 코드는 기능 중심으로 구성한다. 각 기능은 해당 화면, UI 상태, API 연동, 검증 로직을 함께 소유한다.
- API 호출은 화면 렌더링 코드에 흩어 두지 않고, 기능 내부의 명시적인 API 연동 위치에 둔다. 화면은 그 결과 상태를 표시한다.

## 화면 상태와 사용자 입력

모든 비동기 화면은 사용자에게 로딩, 빈 상태, 오류, 권한 없음 상태를 각각 구분해 보여 준다. 오류 상태에는 가능한 범위에서 다음 행동(재시도 또는 이전 화면으로 이동)을 제시한다.

폼은 제출 전과 서버 응답 후에 검증한다. 필수 입력, 형식, 범위, 서버가 반환한 필드 오류를 사용자가 이해할 수 있게 입력 요소 가까이에 표시한다.

## 접근성 기본선

- 의미에 맞는 HTML 요소와 올바른 레이블을 사용한다.
- 키보드만으로 주요 흐름을 조작할 수 있어야 하며, 포커스가 보이고 예측 가능하게 이동해야 한다.
- 상태 변화와 오류는 보조기술에 전달될 수 있게 구현한다.
- 색상만으로 정보나 오류를 전달하지 않는다.

## 후속 선택 사항

전역 상태 도구와 UI 라이브러리는 실제 요구를 확인한 기능 계획에서 선택한다. 이 기준 문서는 특정 도구를 미리 채택하지 않는다.
