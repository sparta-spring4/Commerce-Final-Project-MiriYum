# 기능 명세: 기능 이름

## 관련 정책 ID

- 정책 ID와 정본 정책 문서를 연결한다.

## 사용자 관점의 기능 동작

사용자가 얻는 결과와 기능 경계를 기록한다.

## 권한

역할별로 허용하거나 제한하는 행위를 기록한다.

## 정상 흐름

대표적인 성공 시나리오를 사용자 관점에서 기록한다.

## 오류 흐름

유효하지 않은 입력, 권한 거부, 충돌 등 사용자에게 드러나는 실패 시나리오와 결과를 기록한다.

## API·데이터 계약

요청·응답, 상태값, 필수 데이터와 호환성 경계를 기록한다.

## Migration·호환성 요구

데이터 변경, 이전·전환 순서, 기존 클라이언트 및 데이터와의 호환성 요구를 기록한다.

## 인수 조건

기능이 사용자 행위와 계약을 충족했는지 판단할 수 있는 지속적인 조건을 기록한다.
