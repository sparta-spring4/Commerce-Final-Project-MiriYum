# 광고·추천 사용자 흐름 정합성 수정 설계

## 목적

`docs/04-user-flows.md`의 식당 탐색 흐름을 현재 1차 MVP 범위와 일치시킨다. 일반 검색·추천은 현재 기능으로 유지하고, 후속 범위인 광고 상품·노출·과금·제거 기능은 현재 사용자 흐름에서 활성화하지 않는다.

## 현재 문제

식당 탐색의 주요 흐름은 추천과 광고를 구분해 표시한다고 서술하고 광고 전용 정책인 `ADS-002`, `ADS-006`을 관련 정책으로 참조한다. 이 문구는 광고 기능이 현재 MVP에 포함된 것처럼 읽히지만, `docs/05-functional-requirements.md`와 `docs/service-policies/13-ad-recommendation.md`는 광고 상품·과금·광고 제거를 1차 MVP에서 제외한다.

## 결정

- 식당 탐색의 주요 흐름은 공개 가능한 매장·영업·메뉴 정보를 바탕으로 검색 결과와 일반 추천 결과를 제공한다고 서술한다.
- 관련 정책 ID에서 현재 광고 기능을 전제로 하는 `ADS-002`, `ADS-006`을 제거한다.
- 일반 추천의 품질·설명 규칙인 `ADS-007`, `ADS-008`은 유지한다.
- 광고 안전 정책 자체는 삭제하거나 상태를 변경하지 않는다. 광고 기능이 별도로 활성화될 때 적용하는 후속 정책으로 보존한다.

## 비범위

- 광고 상품, 노출 위치, 과금, 소재 심사 또는 광고 제거 기능을 설계하지 않는다.
- 일반 추천의 랭킹·설명 정책을 변경하지 않는다.
- 이번 변경에서 다른 정합성 문제를 함께 수정하지 않는다.

## 검증 기준

- 식당 탐색 흐름에 현재 광고 노출을 의미하는 문구가 없다.
- 식당 탐색의 관련 정책 ID에 `ADS-002`, `ADS-006`이 없다.
- `ADS-007`, `ADS-008`은 일반 추천 정책으로 계속 참조된다.
- 기능 요구사항과 광고·추천 정책의 1차 MVP 경계 문구는 변경되지 않는다.
- 변경 파일에 UTF-8 BOM이나 여러 개의 끝 개행이 없고 `git diff --check`가 통과한다.
