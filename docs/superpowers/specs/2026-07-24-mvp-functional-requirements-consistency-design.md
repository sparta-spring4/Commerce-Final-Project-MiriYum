# MiriYum MVP 기능 요구사항 정합성 보완 설계

## 목적

PR #19 병합 뒤에도 남아 있는 MVP 제공 범위, 정책 상태, 기능 요구사항 색인의 모순을 제거한다. 이번 변경은 새 제품 기능이나 정책 결정을 추가하지 않고, 이미 확정된 제품 비전과 상세 정책 경계를 요약 문서에 정확히 투영한다.

## 확인된 문제

- `docs/05-functional-requirements.md`는 취소 자리·수량 자동 승계를 `정책 연계·비초기`로 분류하지만 제품 비전과 승계 정책은 이를 초기 제공 범위로 정한다.
- 같은 색인은 일반 추천과 광고, Free 기본 통계와 Pro 분석을 각각 한 행으로 묶고 전체를 비초기로 표시한다.
- 운영·개인정보·보안·신뢰성은 초기 도메인의 공통 구현 기준인데 전체가 비초기로 표시된다.
- PR #19는 `AUTH-004`와 `STORE-014`의 MVP 제외 경계를 정책 마스터에 반영했지만 기능 요구사항 색인에는 전파하지 않았다.
- `docs/service-policies/README.md`의 현재 정책 행을 실제로 집계하면 `확정` 193개, `TODO` 16개이며 중요도는 `핵심` 19개, `필수` 179개, `권장` 18개, `검토` 1개다. 하단 요약과 TODO 목록은 PR #19 이전 값을 유지한다.
- `docs/service-policies/17-privacy-security.md`의 주류 연령 개인정보 절은 `AUTH-004`가 현재 구현되는 것처럼 읽힌다.
- PR #19 변경 파일 일부에 UTF-8 BOM과 여러 개의 끝 개행이 남아 있다.

## 접근안 비교

### 1. 자동 승계 한 셀만 수정

변경량은 가장 작지만 다른 혼합 범위와 공통 기준의 오분류를 그대로 둔다. 같은 유형의 정합성 오류가 남으므로 채택하지 않는다.

### 2. 모든 정책 ID를 MVP 포함·제외 행으로 분해

가장 세밀하지만 217개 정책의 구현 약속처럼 읽히고 상세 정책의 단일 원본 책임을 중복한다. 기능 요구사항 색인의 역할을 넘어가므로 채택하지 않는다.

### 3. 기능 그룹 관계와 상세 경계를 분리

기능 그룹에는 `초기 핵심`, `부분 초기`, `초기 공통 기준`, `비초기` 중 하나를 부여하고, 별도 경계 열에서 포함·제외 또는 상세 정책 참조를 설명한다. 정책 ID의 상태·중요도와 MVP 제공 여부를 분리하면서 색인의 간결성을 유지하므로 이 방식을 채택한다.

## 문서 변경 설계

### 기능 요구사항과 도메인 경계

- `docs/05-functional-requirements.md`
  - `Initial delivery`를 `MVP relation`으로 바꾼다.
  - 네 가지 관계의 의미를 문서 상단에 정의한다.
  - `MVP boundary` 열을 추가해 부분 초기와 공통 기준의 정확한 경계를 기록한다.
  - 취소 자리·수량 자동 승계를 `초기 핵심`으로 바꾼다.
  - 회원·인증, 입점, 예약, 웨이팅, 결제, 체크인, 광고·추천, 분석, 알림을 `부분 초기`로 분류한다.
  - 운영자·개인정보·신뢰성을 `초기 공통 기준`으로 분류하되, TODO나 미래 기능을 활성화하지 않는다고 명시한다.
- `docs/03-domain-model.md`
  - 코스·테이스팅, 구독, 광고 상품, Pro·고급 분석만 비초기로 남긴다.
  - 무료 기본 추천과 Free 기본 통계는 초기 `store` 책임에 포함된다는 점을 명시한다.

### 정책 상태와 개인정보 경계

- `docs/service-policies/README.md`
  - TODO 목록에 `AUTH-004`를 추가한다.
  - 현재 정책 행을 기준으로 상태·중요도 합계를 갱신한다.
  - 다음 단계의 TODO 개수를 16개로 갱신한다.
- `docs/service-policies/17-privacy-security.md`
  - `PRIV-001`의 주류 연령 판정 하위 범위는 주류 기능 도입 시 적용하는 조건부 안전 기준임을 명시한다.
  - 현재 MVP에서 연령 파생값을 생성·저장·조회하지 않는다고 명시한다.
- `miriyum-service-decisions.md`
  - PR #19 이후 현재 상태 스냅샷을 새 변경 기록으로 추가한다.
  - 기존 주류 관련 확정 기록은 미래 도입 시 적용할 정책 방향이고, 현재 행 상태는 `TODO`임을 분명히 한다.

### 인코딩 정리

- PR #19에서 변경됐거나 이번 변경에서 수정하는 정책 파일의 UTF-8 BOM을 제거한다.
- 문서 끝 개행은 하나만 유지한다.
- 의미 변경과 인코딩 정리를 같은 파일에서 구분할 수 있도록 diff를 확인한다.

## 검증 기준

- 기능 요구사항 색인에 취소 자동 승계가 `초기 핵심`으로 표시된다.
- `AUTH-004`와 `STORE-014`가 초기 전체 제공으로 읽히지 않는다.
- 일반 추천·Free 기본 통계와 광고·Pro 분석의 MVP 경계가 분리된다.
- 운영·개인정보·신뢰성은 초기 공통 기준으로 표시되며 미래 기능을 활성화하지 않는다.
- 정책 마스터의 217개 행을 직접 집계한 값과 문서의 현재 요약이 일치한다.
- `AUTH-004`는 TODO 목록과 현재 결정 기록에 포함된다.
- 주류 기능이 도입되지 않은 현재 MVP에서는 연령 파생값을 생성·저장·조회하지 않는다.
- 대상 Markdown 파일은 UTF-8 BOM 없이 끝 개행 하나를 가진다.
- `git diff --check`가 오류 없이 끝난다.
