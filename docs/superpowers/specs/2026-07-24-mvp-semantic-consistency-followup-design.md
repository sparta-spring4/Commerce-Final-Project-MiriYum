# MVP 의미 정합성 후속 수정 설계

## 목적

독립 검수에서 발견된 네 가지 중요 의미 충돌과 사용자가 추가로 바로잡은 리뷰 범위를 현재 1차 MVP 범위와 일치시킨다. 새 기능을 추가하지 않고, 확정된 제품 결정을 제품 비전·권한·도메인·사용자 흐름·기능 요구사항·아키텍처·상세 정책에 정확히 전파한다.

## 확인된 문제

1. 식당 탐색 흐름이 광고 표시와 광고 전용 정책을 현재 기능처럼 참조하지만 광고 상품·과금·제거는 1차 MVP에서 제외된다.
2. 방문 예약 흐름이 승인 대기 상태를 허용하지만 현재 예약 정책은 유효한 요청을 즉시 확정하며 승인 대기 상태를 노출하지 않는다.
3. 결제·취소 흐름이 환불과 자원 복구에서 끝나 초기 핵심인 취소 자리·수량의 FIFO 제안·선점·전환 흐름을 보여주지 않는다.
4. 매장 활성화 기능 목록이 현장 웨이팅만 포함하고 수요 분석 전체를 제외하지만 현재 MVP에는 원격 웨이팅과 Free 기본 운영 통계가 포함된다.
5. 리뷰는 1차 MVP 제외 기능이지만 제품 가치·권한·초기 도메인·사용자 흐름·기능 요구사항·아키텍처와 인증 정책에서 현재 기능처럼 활성화돼 있다.

## 결정

### 일반 추천과 광고 경계

- 식당 탐색의 주요 흐름은 공개 가능한 매장·영업·메뉴 정보를 바탕으로 검색 결과와 일반 추천 결과를 제공한다고 서술한다.
- 관련 정책 ID에서 현재 광고 기능을 전제로 하는 `ADS-002`, `ADS-006`을 제거한다.
- 일반 추천의 품질·설명 규칙인 `ADS-007`, `ADS-008`은 유지한다.
- 광고 안전 정책 자체는 삭제하거나 상태를 변경하지 않는다. 광고 기능이 별도로 활성화될 때 적용하는 후속 정책으로 보존한다.

### 예약 즉시 확정

- 방문 예약의 주요 흐름은 영업시간, 예약 가능 수량과 중복 예약 여부를 검증하고 유효한 요청을 즉시 확정한다고 서술한다.
- 확정할 수 없는 요청은 승인 대기로 남기지 않고 실패 사유를 안내한다.

### 취소 자리·수량 자동 승계

- 기존 결제·취소 흐름은 결제 취소, 환불과 자원 복구의 책임을 유지한다.
- 자동 승계가 활성이고 취소로 반환된 예약 수용량이나 메뉴 수량에 유효한 대기 등록자가 있으면 등록 순번이 가장 빠른 후보에게 5분 제안을 보낸다.
- 수락하면 자원을 10분 동안 선점해 예약·홀드의 검증·결제·확정 흐름으로 전환한다.
- 명시적 거절, 제안 만료, 자격 재검증 실패 또는 검증된 결제 실패에는 정책상 허용되는 다음 후보에게 제안한다. 결제 결과가 불명확하면 공개하거나 다음 후보로 넘기지 않고 대사한다.
- 별도 사용자 흐름에서 `TRANSFER-001`~`TRANSFER-009`를 참조한다.

### 매장 활성화 기능 경계

- 현재 활성화 기능 목록의 현장 웨이팅을 현장·원격 웨이팅으로 고친다.
- Free 기본 운영 통계를 현재 활성화 기능에 포함한다.
- 수요 분석 전체가 아니라 Pro 비교·해석·추천·자동 리포트·내보내기만 후속 범위로 제외한다.
- 현재 지원 기능을 제한하는 문장은 수정된 목록과 연결 정책 경계를 기준으로 정렬한다.

### 리뷰 기능 경계

- 리뷰 작성·공개·수정·삭제, 별점·태그, 사진·텍스트, 신고·숨김·검수, 식당 답글·분쟁, 리뷰 보상·광고 표시, 조작 탐지, 신뢰 점수, 제재와 오탐 복구를 모두 1차 MVP에서 제외한다.
- 제품 비전의 방문 후 경험 평가는 장기 가치로 남기고 초기 비목표에 리뷰 기능을 명시한다.
- 일반 사용자의 현재 권한, 핵심 사용자 흐름, 초기 도메인 모델과 초기 백엔드 패키지에서 `review`를 제거한다.
- 기능 요구사항의 `TRUST-001`~`TRUST-012` 그룹은 `비초기`로 분류한다.
- 리뷰 정책의 세부 결정과 행 단위 `확정` 상태는 삭제하지 않는다. 향후 리뷰 기능 도입 시 적용할 정책 기준으로 보존하되 현재 기능·권한·API·UI·저장소를 활성화하지 않는다고 명시한다.
- 회원·인증 정책에서 만 14세 이상 사용자가 현재 리뷰를 이용할 수 있다는 문구를 제거하되, 미래 로그인 필수 기능의 연령·계정 안전 기준은 보존한다.

## 비범위

- 광고 상품, 노출 위치, 과금, 소재 심사 또는 광고 제거 기능을 설계하지 않는다.
- 일반 추천의 랭킹·설명 정책을 변경하지 않는다.
- 예약 승인 워크플로를 추가하지 않는다.
- 자동 승계 정책의 FIFO, 제한 시간, 선점 또는 전환 규칙을 새로 결정하지 않는다.
- Pro 분석 기능을 현재 MVP로 활성화하지 않는다.
- 독립 검수에서 확인된 경미한 표기·인코딩 문제는 이번 변경에 포함하지 않는다.
- 리뷰 상세 정책을 폐기하거나 정책 ID의 상태·중요도를 변경하지 않는다.

## 검증 기준

- 식당 탐색 흐름에 현재 광고 노출을 의미하는 문구가 없다.
- 식당 탐색의 관련 정책 ID에 `ADS-002`, `ADS-006`이 없다.
- `ADS-007`, `ADS-008`은 일반 추천 정책으로 계속 참조된다.
- 방문 예약 흐름에 승인 대기 상태가 없고 즉시 확정 및 실패 안내가 명시된다.
- 취소 자리·수량 자동 승계 흐름이 `TRANSFER-001`~`TRANSFER-009`를 참조하고 FIFO 제안·5분 응답·10분 선점·전환·검증된 실패의 다음 대상 진행·결제 결과 불명확 시 대사를 설명한다.
- 매장 활성화 기능 목록이 현장·원격 웨이팅과 Free 기본 운영 통계를 포함한다.
- 후속 분석 범위는 Pro 비교·해석·추천·자동 리포트·내보내기로 한정된다.
- 제품 비전의 초기 비목표와 기능 요구사항에 리뷰 기능 전체가 1차 MVP 제외로 표시된다.
- 현재 사용자 권한·핵심 사용자 흐름·초기 도메인 모델·초기 아키텍처에 `review` 기능이나 모듈이 없다.
- 회원·인증 정책이 리뷰를 현재 이용 가능한 기능으로 서술하지 않는다.
- 리뷰 상세 정책과 정책 마스터는 정책 확정과 현재 MVP 활성화를 구분하고, 현재 리뷰 기능·권한·API·UI·저장소를 활성화하지 않는다.
- 현재 경계 변경이 결정 기록에 추가된다.
- 최초 블루프린트는 역사적 입력임을 파일 자체에서 명시하고 현재 정본보다 우선하지 않는다.
- 기존 네 정합성 수정의 기능 요구사항·상세 정책 경계는 유지되고, 리뷰 범위만 사용자의 현재 제외 결정에 맞게 정렬된다.
- 변경 파일에 UTF-8 BOM이나 여러 개의 끝 개행이 없고 `git diff --check`가 통과한다.
