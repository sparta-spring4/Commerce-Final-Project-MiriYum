# 현장·원격 웨이팅 정책

> 문서 상태: 논의 중
> 정책 범위: `WAIT-001`~`WAIT-017`
> 최종 변경일: 2026-07-21

## 1. 목적

사용자가 매장 근처에서 원격 웨이팅을 등록하고 일행과 대기 상태를 공유할 수 있게 하면서, 같은 일행이 여러 계정으로 중복 등록해 허수 대기 팀을 만드는 문제를 줄인다.

## 2. 확정된 정책

### WAIT-002 대표자 위치 검증

- 웨이팅 대표자는 매장이 허용한 반경 안에 있어야 신규 대기 팀을 만들 수 있다.
- 위치 정보는 등록 가능 여부 확인에 필요한 최소 범위로 사용한다.
- 정확한 원본 좌표는 불필요하게 장기 보관하지 않는다.

### WAIT-005 일행 초대와 거리 제한

- 대표자는 일행 참여 링크 또는 초대 코드를 공유할 수 있다.
- 초대받은 일행은 거리 제한 없이 기존 대기 팀에 참여할 수 있다.
- 일행 참여는 새로운 대기 팀을 만들지 않는다.
- 참여한 일행은 같은 순번과 예상 대기 시간을 확인한다.

## 3. 팀 협의 필요 사항

### WAIT-003 등록 반경

선택지:

1. 모든 매장에 고정 반경을 적용한다.
2. 매장이 반경을 자유롭게 설정한다.
3. 플랫폼이 최소·최대 범위를 정하고 매장이 그 안에서 선택한다.

현재 추천은 3번이다. 예시 기본값 300m와 설정 범위 100m~1km는 확정값이 아니며 실제 GPS 오차와 상권 테스트 후 결정한다.

### WAIT-004 위치 검증 예외

- 실내·지하·고층 건물에서 GPS 정확도가 낮을 때의 대체 방식
- 위치 권한을 거부한 사용자의 현장 QR 또는 매장 코드 등록 허용 여부
- 위치 조작이 의심될 때 추가 확인 방식
- 매장 좌표가 잘못되었을 때 운영자 보정 절차

## 4. 검토 중인 보완 정책

다음 내용은 제안 상태이며 확정되지 않았다.

- 가입 시 휴대전화 번호를 한 번 확인한다.
- 한 계정은 같은 매장과 겹치는 시간대에 대기 팀을 하나만 보유한다.
- 기존 대기 팀의 일행은 같은 매장에 별도 대기 팀을 만들 수 없다.
- 차례가 가까워지면 대기 지속 의사를 확인하고 미응답 시 취소한다.
- 반복 미응답·노쇼에는 원격 웨이팅 제한 또는 일시 정지를 적용한다.
- 필요한 인기 매장에는 체크인 시 반환되는 소액 보증금을 허용한다.
- 중복 가능성을 탐지하되 자동 탐지 결과만으로 즉시 제재하지 않는다.

## 5. 아직 결정하지 않은 핵심 항목

- 일행 합류·이탈·대표자 변경
- 사용자별 동시 웨이팅 허용 수
- 대기 순번 산정과 매장 조정 권한
- 예상 대기 시간 계산과 정확도 표시
- 호출 횟수, 응답 시간, 늦은 도착 및 재등록
- 허수 웨이팅 탐지 기준, 제재, 회복 및 이의 제기
- 상태 전이와 다중 인스턴스 순번 정합성
