# TasteLock 서비스 정책 마스터 체크리스트

> 문서 상태: 운영 중
> 기준일: 2026-07-21
> 목적: 구현 전에 결정해야 할 정책을 빠짐없이 추적한다.

## 상태 정의

- `미논의`: 아직 선택지와 영향을 검토하지 않음
- `논의 중`: 현재 대화에서 검토 중
- `팀 협의 필요`: 팀의 공동 결정이 필요함
- `확정`: 사용자가 명시적으로 승인함
- `변경 검토`: 기존 확정 내용을 다시 검토 중

## 운영 규칙

- 정책 ID는 이름이 바뀌어도 변경하지 않는다.
- 한 번에 하나의 정책 질문만 확정한다.
- 추천안과 확정안을 구분한다.
- 상세 정책이 끝나면 예외, 악용, 동시성, 장애 및 테스트 관점으로 재검토한다.
- 제품 범위 변경은 `docs/service-definition.md`, 기술 방향 변경은 `docs/technical-architecture.md`에 반영한다.
- 결정 및 변경 이력은 `tastelock-service-decisions.md`에 남긴다.
