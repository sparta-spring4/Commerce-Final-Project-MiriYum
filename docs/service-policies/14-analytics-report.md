# 분석·수요 리포트 정책

> **1차 MVP 경계:** 매장 Free에는 아래 기본 운영 통계를 제공한다. 상권 비교, 전주·전월 대비 해석, 취소 원인 분석, 운영 추천과 자동 리포트는 향후 매장 Pro 후보이며 현재 권한이나 필수 구현이 아니다.

> 문서 상태: 세부 정책 확정
> 정책 범위: `ANALYTICS-001`~`ANALYTICS-009`
> 최종 변경일: 2026-07-22

행동 이벤트, 수요 지표, 미충족 수요, 익명화, 상권 비교, 데이터 품질 및 재집계를 다룬다. 분석은 기존 운영 상태를 일관되게 집계하는 파생 기능이며 새로운 추적·개인정보 수집의 근거가 아니다.

|---|---|---|
| 확정 | 필수 | `ANALYTICS-001`~`ANALYTICS-003`, `ANALYTICS-005`~`ANALYTICS-007`, `ANALYTICS-009` |

## 분석 공통 불변식

- MySQL의 중앙 운영 상태와 각 도메인의 Outbox 사건이 원본이다. 분석을 위해 새 페이지 추적, 광고 식별자, 정밀 위치, 기기 지문, 연락처, 자유 텍스트 또는 민감정보를 수집하지 않는다.
- 분석 이벤트는 원본 사건 ID·집계 대상의 비식별 내부 ID·도메인 버전·사건 시각·스키마 버전과 지표에 필요한 최소 상태만 가진다. 결제수단 원문, 리뷰 원문, 동행자·신고자 신원은 복제하지 않는다.
- 집계는 정의 버전·입력 체크포인트·출력 스냅샷을 기록하고 같은 입력에는 같은 결과를 낸다. 중복·역순·지연·정정·삭제 사건을 정상 조건으로 취급하며 단일 작업 인스턴스나 처리 순서에 의존하지 않는다.

## ANALYTICS-001 수집할 행동 이벤트

> 정책 상태: 확정

### 확정 이벤트 범위·권한·상태

- 수집 대상은 계정·매장·영업, 예약·웨이팅·홀드·테이스팅, 체크인·노쇼, 승계, 결제 상태, 구독 혜택, 리뷰·신고, 알림처럼 서비스 수행 과정에서 이미 중앙 상태 변경으로 발생하는 운영 사건으로 한정한다. 화면 조회·스크롤·마우스 이동 같은 새 행동 추적은 기본 수집하지 않는다.
- 각 도메인만 자신의 확정 트랜잭션과 함께 Outbox 사건을 만들 수 있고 분석 작업자는 원본 상태를 수정하지 않는다. 운영자 수동 입력으로 존재하지 않은 사용자 행동을 생성하지 못하며 정정은 원 도메인의 보정 사건을 따른다.
- 이벤트는 `발행 대기`, `수신`, `검증`, `격리`, `반영`, `정정`, `삭제 반영` 상태를 가진다. 필수 필드는 사건 ID, 도메인·유형, 집계 대상 내부 ID, 원본 버전, 발생 시각, 스키마 버전과 최소 측정값이다.

### 동시성·실패·감사·개인정보

- 사건 ID와 원본 버전의 유일성으로 중복 수신을 제거하고 Outbox 재전송·다중 소비자에도 한 번만 반영한다. 알 수 없는 스키마·필수 필드 누락은 추정 보완하지 않고 격리 후 재처리한다.
- 발행 지연·브로커 장애는 원 거래를 되돌리지 않고 Outbox 체크포인트에서 재개한다. 사건 유형·스키마·생산 도메인·격리·재처리 결과를 감사·관측하되 원문 PII를 로그에 남기지 않는다.
- 새 이벤트나 필드는 목적·지표·최소성 검토와 스키마 버전 승인을 거쳐야 하며 기존 개인정보를 분석 편의상 복제하는 변경은 거부한다.

### 관측 가능한 인수 조건

- 기존 운영 상태 변경 없이 화면 탐색만 한 행위는 분석 이벤트를 새로 만들지 않는다.
- 동일 Outbox 사건을 병렬·반복 소비해도 입력 원장과 지표 반영은 하나다.
- 이벤트 스키마에 연락처·리뷰 원문·결제수단·정밀 위치·기기 지문이 없고 알 수 없는 사건은 격리된다.

## ANALYTICS-002 지표 정의와 중복 제거

> 정책 상태: 확정

### 확정 정의·계산·게시

- 모든 지표는 이름, 목적, 포함·제외 사건, 분자·분모, 상태 판정, 사건 시각 기준, 시간대, 집계 차원, 지연 사건 처리와 정의 버전을 갖는다. 정의가 없거나 버전이 다른 값을 같은 이름으로 합치지 않는다.
- 사건 ID-지표 정의 버전별 한 번만 반영하고 상태형 지표는 원본 집계 ID의 최신 유효 버전을 사용한다. 재시도·중복 웹훅·동일 거래의 여러 기술 사건을 별도 사용자 수요나 성공으로 중복 세지 않는다.
- 지표 스냅샷에는 정의 버전, 입력 체크포인트, 집계 범위, 생성 시각, 데이터 기준 시각과 보정 여부를 함께 게시한다. 정의 변경은 새 버전으로 재집계하고 과거 값과 비교할 때 단절을 명시한다.

### 동시성·실패·정정·감사·개인정보

- 분할 작업은 결정적 파티션 키와 중앙 체크포인트를 사용하고 출력 키-정의 버전-기간 유일성으로 중복 게시를 막는다. 완료되지 않은 부분 집계를 최종 값으로 노출하지 않는다.
- 늦은·역순 사건과 원본 정정은 보정 원장으로 반영해 같은 입력 집합이면 전체 재실행과 증분 결과가 같게 한다. 오류 스냅샷은 삭제 덮어쓰기보다 대체 버전과 사유를 남긴다.
- 정의 승인·입력 체크포인트·중복 제거·보정·게시를 감사하고 개인별 원문 목록을 리포트에 노출하지 않는다.

### 관측 가능한 인수 조건

- 같은 입력과 정의 버전을 반복·병렬 집계해도 동일한 지표 값과 스냅샷 무결성 값이 나온다.
- 중복 웹훅·재시도 사건이 분자·분모를 늘리지 않고 늦은 정정 뒤 증분·전체 재집계 값이 일치한다.
- 모든 표시 지표에서 정의·기준 시각·집계 범위·보정 여부를 조회할 수 있다.

## ANALYTICS-003 미충족 수요 추정 방법

> 정책 상태: 확정

### 확정 입력·추정·표시 경계

- 미충족 수요는 기존 운영 사건 중 가용성 부족으로 검증된 예약·홀드·회차 요청 거절, 활성 대기/승계 등록, 품절·마감 상태처럼 실제 수요 의사가 상태로 남은 사건에서만 추정한다. 단순 페이지 조회·검색·광고 클릭을 수요로 간주하지 않는다.
- 같은 사용자·대상·의도 구간의 재시도와 기술 중복은 하나의 수요 시도로 중복 제거하고, 자격 미달·입력 오류·결제 결과 불명확·어뷰징 확정 사건은 가용성 부족과 분리한다. 사용자를 교차 서비스에서 추적하거나 기기 지문으로 동일인을 추정하지 않는다.
- 결과는 `관측된 미충족 시도`와 제한된 추정치로 표시하고 입력 범위·정의 버전·누락 가능성·데이터 기준 시각을 함께 제공한다. 실제 구매 의향, 미래 매출 또는 개인별 선호로 단정하지 않는다.

### 동시성·실패·재추정·감사·개인정보

- 의도 키·원 사건 ID와 정의 버전으로 중복을 제거하고 늦은 성공·취소·자격 정정은 기존 미충족 분류를 보정한다. 집계 원본 장애에는 추정치를 이전 값으로 조용히 유지하지 않고 지연·불완전 상태를 표시한다.
- 정의 변경·원본 정정에는 영향 기간을 멱등 재추정하고 기존 공개 스냅샷을 대체 버전으로 남긴다. 입력 분류·제외·보정·재추정을 감사한다.
- 사용자 식별자는 집계 전에 내부 처리 범위에서만 사용하고 출력은 `ANALYTICS-006` 기준으로 억제·집계한다. 연락처·정밀 위치·결제 금액 원문을 수요 추정에 추가하지 않는다.

### 관측 가능한 인수 조건

- 조회·새로고침·광고 클릭만으로 미충족 수요가 증가하지 않고 같은 거절의 재시도는 한 의도로 계산된다.
- 늦은 성공·자격 정정 후 해당 시도가 미충족 집계에서 보정되고 전체 재집계와 일치한다.
- 리포트가 추정의 입력·한계·기준 시각을 표시하며 개인별 수요 목록이나 새 PII를 노출하지 않는다.

## ANALYTICS-004 리포트 제공 지표

> 정책 상태: 확정

### 확정 제공 범위·권한·스냅샷

- 매장 Free의 1차 MVP 기본 통계는 `오늘 예약 팀 수`, `예약률`, `팀 수 상한 사용률`, `취소율`, `웨이팅 현황`, `노쇼 현황`이다. 이 권한은 구독이나 Pro 출시와 무관하며 유효한 매장 소속을 가진 식당 대표자가 자기 매장 범위에서 조회한다.
- `오늘 예약 팀 수`는 매장 시간대 오늘 방문 예정인 확정·체크인·이행 완료·노쇼 후보·노쇼 확정 예약 팀 수다. 취소·거절·결제 실패·만료는 제외한다.
- `예약률`은 오늘 서비스 구간별 확정 예약 인원 점유량을 플랫폼 예약 제공 인원으로 나눈 값의 합산 비율이고, `팀 수 상한 사용률`은 확정 예약 팀 수를 최대 팀 수로 나눈 값의 합산 비율이다. 분모가 없거나 유효하지 않은 구간은 0으로 꾸미지 않고 산출 불가로 표시한다.
- `취소율`은 해당 기간에 방문 예정이었던 확정 예약 중 취소된 예약 팀 수 비율이다. `웨이팅 현황`은 현재 대기 중·호출 중 팀 수와 인원수·가장 오래 기다린 시간, `노쇼 현황`은 노쇼 후보와 노쇼 확정을 분리한 팀 수다.
- 결제 실패로 확정되지 않은 거래는 예약·취소 통계에 넣지 않는다. 예약 변경은 최신 유효 버전만 사용하고, 사건 ID로 중복 반영을 막으며 정정·삭제 사건을 다음 재집계에 반영한다.
- 상권 비교, 전주·전월 대비 해석, 취소 원인 분석, 운영 추천, 자동·주간·월간 리포트와 내보내기는 향후 Pro 후보다. `SUB-001`~`SUB-003`의 유효한 버전 권한과 출시 결정 전에는 생성·노출하지 않는다.
- 각 기본 통계 스냅샷은 매장 권한 버전, 지표 정의 버전, 대상 매장·기간, 생성 시각, 마지막 반영 사건 시각, 데이터 기준 시각, 지연·보정·격리 상태를 함께 고정한다. 집계 지연·격리 중에는 정상 최신 수치처럼 표시하지 않는다.

### 동시성·실패·알림·감사·개인정보

- 기본 통계는 중앙 매장 소속·권한을, 향후 Pro 리포트는 매장 권한과 `SUB-001`~`SUB-003` 권한 버전을 함께 재검증한다. 스냅샷 ID-매장 권한-정의 버전-기간의 유일성과 생성 멱등 키로 다중 인스턴스의 중복 게시를 막는다.
- 매장 소속이 회수되면 기본·고급 통계의 가시성·캐시를 차단한다. Pro 권한만 회수·만료되면 고급 리포트·내보내기를 차단하되 매장 Free 기본 통계 권한까지 제거하지 않는다.
- 권한 원본 장애에는 이전 캐시나 화면 상태를 근거로 조회·내보내기를 허용하지 않는다. 집계 실패·지연에는 마지막 반영 사건 시각과 생성 시각, 불완전 상태를 표시하고 부분 값을 최신 확정으로 가장하지 않는다.
- 리포트 권한 판정·생성·조회·내보내기·거부·회수·보정을 감사하고 필요시 데이터 품질 경보를 운영자에게 알린다. 권한 확인과 스냅샷에는 개인별 행, 연락처, 리뷰 원문, 결제수단·신고자 정보를 포함하지 않고 최소 집계와 억제 규칙을 적용한다.

### 관측 가능한 인수 조건

- 유효한 매장 소속만 있으면 구독 없이 정확히 여섯 기본 지표를 조회할 수 있고 다른 매장이나 개인별 원자료는 볼 수 없다.
- Pro 권한이 없으면 비교·해석·추천·자동 리포트와 내보내기가 생성·노출되지 않지만 기본 통계는 계속 제공된다.
- 중복·역순 사건과 예약 정정·삭제 뒤 증분 집계와 전체 재집계가 일치하고 결제 실패 거래가 예약·취소 지표에 들어가지 않는다.
- 화면의 마지막 반영 사건 시각·생성 시각·지연/격리 상태가 실제 스냅샷과 일치하고 부분·오래된 값을 정상 최신 수치로 표시하지 않는다.

## ANALYTICS-005 집계 주기와 지연 표시

> 정책 상태: 확정

### 확정 실행·상태·게시 흐름

- 집계는 지표별 버전된 실행 정책과 사건 시각을 사용해 증분 처리하고, 정확한 주기·허용 지연 수치는 관측 후 조정 가능한 운영 설정으로 둔다. 이 정책은 근거 없는 SLO·시간 숫자를 확정하지 않는다.
- 작업 상태는 `대기`, `실행 중`, `검증`, `게시`, `지연`, `실패`, `재처리`로 관리한다. 새 스냅샷은 입력 체크포인트까지 완전 처리하고 품질 검증을 통과한 뒤에만 원자 게시한다.
- 화면에는 데이터 기준 시각, 생성 시각, 현재 지연·재처리 여부를 항상 표시한다. 지연 중에는 오래된 스냅샷을 최신처럼 표시하지 않고 기준 시각을 유지한다.

### 다중 인스턴스·실패·복구·감사

- 작업 임대 또는 중앙 작업 테이블, 파티션 체크포인트와 출력 버전 유일성으로 여러 작업자가 같은 범위를 중복 게시하지 않는다. 임대 만료 뒤 다른 작업자가 이어받아도 처리 자체는 멱등하다.
- 브로커·DB·작업 장애에는 완료 체크포인트부터 재개하고 부분 출력은 비공개로 폐기 또는 재사용한다. 늦은 사건은 정의된 보정 창과 재집계 작업으로 처리하되 성공 스냅샷을 제자리 덮어쓰지 않는다.
- 작업 소유권·체크포인트·지연·실패·재개·게시를 감사하고 지연·연속 실패를 운영자 경보로 연결한다. 체크포인트·작업 로그에는 개인 식별자나 사건 원문 대신 파티션·버전·집계 범위만 남긴다. 경보 임계치는 실측 운영 설정이며 여기서 숫자를 정하지 않는다.

### 관측 가능한 인수 조건

- 두 작업 인스턴스가 같은 기간을 실행해도 게시 스냅샷은 하나이고 중복 합산되지 않는다.
- 중단·임대 만료·재시작 뒤 완전한 체크포인트에서 수렴하며 부분 집계가 최신 값으로 표시되지 않는다.
- 모든 리포트에 데이터 기준 시각과 지연·재처리 상태가 있어 사용자가 최신성을 오해하지 않는다.

## ANALYTICS-006 익명화와 최소 집단 크기

> 정책 상태: 확정

### 확정 최소화·억제·권한 경계

- 리포트 출력은 개인 식별자와 자유 텍스트를 제거한 집계만 제공한다. 단순 해시·닉네임 제거를 익명화로 간주하지 않고, 희소 조합·세부 시간/지역 구간·반복 질의로 개인이나 특정 거래를 재식별할 수 있는 결과를 억제하거나 더 큰 구간으로 병합한다.
- 교차 매장·상권 비교와 외부 공유에는 중앙 개인정보 정책 버전의 최소 집단 기준을 통과한 셀만 게시한다. 기준이 아직 검증되지 않았거나 집단 크기를 계산할 수 없으면 세부 셀을 실패 폐쇄하고 비교 결과를 제공하지 않는다. 이 문서에서 근거 없는 숫자를 정하지 않는다.
- 자기 매장 운영 리포트도 개인별 목록 대신 집계로 제공하며 매우 작은 셀은 상위 기간·카테고리로 묶는다. 권한 있는 운영자의 원본 사건 접근은 별도 업무 권한과 감사 대상이지 분석 리포트 우회가 아니다.

### 동시성·정정·차분 공격 방지·감사

- 억제는 지표 정의·개인정보 정책·집계 스냅샷 버전에 결정적으로 적용하고 출력 키 유일성과 생성 멱등 키로 다중 인스턴스의 캐시·내보내기에도 동일한 결과를 사용한다. 정정·삭제로 집단이 기준 아래가 되면 새 스냅샷에서 즉시 억제하고 기존 캐시를 무효화한다.
- 인접 기간·필터 조합으로 억제 셀을 차분 복원하지 못하도록 제공 가능한 차원과 세분화를 버전 정책으로 제한한다. 반복 질의·내보내기 권한을 중앙에서 통제하되 사용자 추적용 새 식별자를 수집하지 않는다.
- 정책 버전·억제/병합 사유·조회·내보내기·캐시 무효화를 감사하고 출력에서 내부 사용자·거래 ID를 제거한다.

### 관측 가능한 인수 조건

- 최소 집단 기준 미확정·미충족·계산 실패 시 교차 매장 세부 셀이 공개되지 않는다.
- 화면·API·내보내기와 여러 인스턴스가 동일 스냅샷에 같은 억제 결과를 적용한다.
- 삭제로 셀이 희소해지면 캐시 포함 파생 결과가 억제되고 필터 조합으로 개인별 값을 복원할 수 없다.

## ANALYTICS-007 데이터 정정·삭제 반영

> 정책 상태: 확정

### 확정 전파·계보·재처리

- 원 도메인의 정정, 삭제, 익명화와 법적 보존 분리 사건을 분석의 권위 있는 변경 입력으로 사용한다. 분석 작업자는 원본 내용을 임의 복구하거나 삭제 요청을 성공으로 추정하지 않는다.
- 각 파생 데이터는 원 사건 ID·버전, 지표 정의, 집계 스냅샷과 출력 위치의 계보를 가져 영향 범위를 찾는다. 정정은 보정 사건으로 더하고 삭제는 개인 식별 연결·세부 원문을 제거한 뒤 집계를 다시 계산한다.
- 전파 상태는 `접수`, `영향 분석`, `파생 삭제/보정`, `재집계`, `검증`, `완료`, `실패`로 관리한다. 검색·캐시·내보내기용 스냅샷도 같은 사건을 소비하고 완료 전까지 오래된 세부 결과를 숨기거나 지연 상태로 표시한다.

### 동시성·실패·복구·감사·개인정보

- 원 변경 사건-파생 대상별 멱등 키와 체크포인트로 중복 삭제·중복 보정을 막는다. 일부 대상 실패는 완료 대상을 되돌리지 않고 실패 대상만 재처리하며 전체 완료는 모든 필수 파생 대상 검증 뒤 확정한다.
- 백업 복원·전체 재집계도 삭제 원장과 보정 사건을 다시 적용해 삭제된 PII가 서비스 파생물에 부활하지 않게 한다. 보존 기간·법적 예외는 PRIV TODO를 따르고 분석이 독자 결정하지 않는다.
- 요청자에게 제공되는 상태가 있다면 처리 범위와 완료만 알리고 내부 데이터 위치는 노출하지 않는다. 원 사건·영향 대상·작업자·실패·재처리·검증을 감사한다.

### 관측 가능한 인수 조건

- 같은 삭제 사건을 반복 처리해도 집계가 여러 번 감소하지 않고 모든 필수 파생 대상이 완료 상태로 수렴한다.
- 작업 중단·백업 복원·전체 재집계 뒤에도 삭제된 식별 연결·원문이 검색·캐시·리포트에 다시 나타나지 않는다.
- 정정 전후 지표 차이가 보정 사건과 계보로 재현되고 미완료 파생 대상이 관측·재처리된다.

## ANALYTICS-008 상권 비교 기준과 공정성

> 정책 상태: 확정

### 확정 비교 가능성·표시 경계

- 비교는 동일한 지표 정의 버전, 시간대·기간, 업종/카테고리, 운영 기능 범위와 데이터 완전성 기준을 충족한 집단끼리만 수행한다. 지역 경계는 공개 매장 주소에서 이미 운영에 사용하는 행정·상권 구분의 버전만 사용하고 사용자 정밀 위치를 추가 수집하지 않는다.
- 표본·기능 사용·휴점·신규 입점 차이를 함께 표시하고 비교 불가능한 집단은 순위나 우열 값으로 만들지 않는다. `ANALYTICS-006` 최소 집단 기준 미충족 셀은 병합 또는 억제한다.
- 비교 결과는 설명·운영 참고용이며 광고 노출, 구독 혜택, 제재, 수수료, 검색 순위나 희소 자원 배분을 자동 변경하지 않는다. 특정 매장의 성과 보장이나 사업적 불이익을 이 정책에서 만들지 않는다.

### 동시성·정정·재비교·감사·개인정보

- 비교군 정의·멤버십·지표 스냅샷을 같은 버전으로 고정하고 원자 게시한다. 매장 분류·주소·영업 상태 정정은 새 비교 스냅샷을 멱등 재계산하고 과거 기준을 소급 덮어쓰지 않는다.
- 데이터 누락·비교군 편중·정의 버전 불일치는 결과를 보류하고 사유를 표시한다. 비교군 정의·제외·억제·재계산·조회·내보내기를 감사한다.
- 개인·거래 단위 데이터나 매장 비공개 세부를 다른 매장에 노출하지 않고 비식별 집계만 사용한다.

### 관측 가능한 인수 조건

- 정의·기간·기능 범위가 다른 집단은 하나의 순위로 비교되지 않고 불일치 사유가 표시된다.
- 최소 집단 미충족·데이터 누락에는 세부 비교가 억제되며 결과가 광고·제재·가격·자원 배분에 입력되지 않는다.
- 분류 정정·재집계 뒤 같은 비교군/지표 버전은 결정적으로 같은 결과를 내고 과거 스냅샷의 기준도 추적된다.

## ANALYTICS-009 데이터 품질과 재집계

> 정책 상태: 확정

### 확정 품질 검사·상태·게시

- 품질 검사는 스키마 적합성, 사건 ID 유일성, 원본 참조·버전 유효성, 상태 전이, 필수 필드, 사건 시각, 집계 합계 불변식과 정정·삭제 적용 여부를 지표별로 수행한다. 검사를 통과하지 못한 입력은 격리하고 원문을 임의 보정하지 않는다.
- 데이터셋·기간 상태는 `정상`, `지연`, `불완전`, `격리`, `재집계 중`, `검증 실패`, `게시 가능`으로 관리한다. 게시 가능 스냅샷은 정의 버전·입력 체크포인트·품질 결과·무결성 값을 함께 고정한다.
- 재집계는 입력 스냅샷과 정의 버전을 고정해 별도 출력으로 만들고 품질 검증 후 기존 스냅샷을 원자 교체한다. 실패하면 마지막 정상 스냅샷과 그 기준 시각을 유지하고 부분 결과를 노출하지 않는다.

### 다중 인스턴스·실패·복구·감사·개인정보

- 작업 범위별 중앙 임대·체크포인트, 입력 사건 중복 제거와 출력 유일성으로 여러 작업자의 중복 합산·동시 교체를 막는다. 작업 인스턴스 유실 뒤 다른 작업자가 이어받아도 같은 결과를 만든다.
- 격리 입력은 생산 도메인·스키마 소유자에게 경보하고 수정 사건 뒤 재처리한다. 품질 규칙 변경은 고정 검증 데이터로 확인하고 이전 정상 결과와 차이를 기록한 뒤 배포한다.
- 검사·격리·재처리·재집계·교체·롤백과 운영자 개입을 감사한다. 품질 로그에는 이벤트 원문·PII 대신 사건 ID, 스키마·오류 코드와 무결성 참조만 남긴다.

### 관측 가능한 인수 조건

- 중복·누락·역순·잘못된 스키마를 주입하면 해당 입력/스냅샷이 게시되지 않고 격리·경보가 생성된다.
- 같은 입력·정의 버전의 증분 처리와 전체 재집계가 동일하며 병렬 작업에도 출력 스냅샷 하나만 게시된다.
- 재집계 실패 시 부분 결과가 노출되지 않고 마지막 정상 값의 기준 시각이 표시되며 수정 후 멱등 재처리로 수렴한다.
