# 광고·추천 정책

> **1차 MVP 경계:** 일반 검색·자연어·취향·품절 대체 추천은 무료 기본 기능이다. 광고 상품·광고 과금·광고 제거는 1차 MVP에서 제외하며, 아래 광고 안전 정책이 현재 광고 기능이나 권한을 활성화하지 않는다.

> 문서 상태: 세부 정책 확정(팀원 상의 필요·TODO 및 확정 하위 범위 보존)
> 정책 범위: `ADS-001`~`ADS-008`
> 최종 변경일: 2026-07-22

광고 상품, 표시, 과금, 소재 심사, 타게팅, 빈도 제한 및 추천 랭킹을 다룬다. 광고와 일반 추천을 명확히 구분하는 원칙은 확정되었다.

## 분류 현황

| 상태 | 중요도 | 정책 ID |
|---|---|---|
| 확정 | 필수 | `ADS-002` |
| 팀원 상의 필요 | 핵심 | `ADS-001` |
| 확정 | 필수 | `ADS-007` |
| 확정 | 권장 | `ADS-006`, `ADS-008` |
| TODO | 필수 | `ADS-003`~`ADS-005` |

- `ADS-001`: 광고 상품·노출 위치는 수익모델과 사용자 경험의 경계를 정하므로 팀 합의가 필요하다.
- `ADS-003`: 과금 방식과 무효 트래픽 정산은 광고 사업자 계약·수익모델 확정 후 결정한다.
- `ADS-004`: 미성년자 주류 광고 차단과 안전 지면 사용은 확정으로 보존한다. TODO는 일반·주류 소재 전체의 최종 심사 기준과 외부 사업자 기능·법률 검토에 한정한다.
- `ADS-005`: 생년월일·주류 이용 가능일을 외부에 전달하지 않고 내부 연령 판정으로 지면을 선택하는 하위 범위는 확정으로 보존한다. TODO는 타게팅 데이터 전체 목록·동의·보관·프로파일링·외부 제공의 계약·법률 검토에 한정한다.
- `ADS-006`은 광고 상품이 별도로 확정된 경우에만 적용하는 빈도·숨김 안전 제어이고, `ADS-007`·`ADS-008`은 유료 광고와 분리된 일반 추천의 품질·설명 규칙이다.

## 1차 MVP 일반 검색·추천 계약

### 검색·정렬·페이징

- 매장명·메뉴명·지역명 키워드와 지역, 매장·메뉴 카테고리, 예약 가능 여부를 검색한다. 서로 다른 필터는 `AND`, 같은 필터의 복수 값은 `OR`로 결합한다.
- 커서 페이징은 기본 20개·최대 50개이며 필터나 정렬을 바꾸면 기존 커서를 폐기한다. 키워드는 관련도, 자연어 추천은 추천 점수, 주변 검색은 거리, 예약 시간 검색은 가장 빠른 가능 시간을 기본 정렬로 사용한다.
- 동률은 주 카테고리 일치, 보조 카테고리 일치 수, 실제 예약 가능 여부, 거리, 매장 ID 순으로 결정한다. 표시 직전과 예약·홀드 확정 직전에 MySQL 중앙 원본의 영업·품절·수용량을 다시 검증한다.

### 자연어 조건 해석·취향 재정렬

- `자연어 조건 해석기`는 MiriYum 백엔드 구성요소다. 숫자·단위·시각을 정규화하고 MiriYum 지역·카테고리·태그 사전과 결정적 규칙으로 명확한 조건을 먼저 추출한다.
- 규칙으로 해석하지 못한 표현만 개발·시연용 Spring AI OpenAI 어댑터에 전달하고, 반환값은 허용된 MiriYum 코드와 범위로 검증한 뒤 OpenSearch를 조회한다. AI 실패·시간 초과·형식 오류에는 규칙 결과만 사용하고, 규칙 결과도 없으면 입력 문장을 일반 키워드 검색으로 넘긴다.
- OpenSearch는 조건 추출기가 아니라 Nori 기반 한국어 일반 검색, `geo_point` 주변 검색과 벡터 후보 검색을 담당한다. Meilisearch를 추가하지 않는다.
- 이전 예약과 최근 메뉴 홀드 이력은 외부 AI로 보내지 않고 MiriYum 내부 최종 재정렬에만 사용한다. 외부 AI에는 사용자 입력 문장과 허용 코드만 전달하고 사용자 ID·연락처·예약 원문·홀드 이력·결제정보·정확한 위치를 전달하지 않으며 원문 프롬프트·응답을 운영 로그에 보관하지 않는다.

### 위치·알레르기·품절 대체

- 현재 위치는 사용자가 주변·거리 기능을 명시적으로 실행할 때만 요청하고 원 좌표는 요청 중에만 사용한다. 권한 거부·오류에는 시·군·구 또는 동 직접 선택을 제공하며 공용 캐시는 익명화 지역 셀을 사용한다.
- 현재 요청의 알레르기 제외 조건을 위반하거나 `함유`·`혼입 가능`·`정보 미등록` 상태로 위험을 배제할 수 없는 후보는 안전 우선 추천에서 제외하고 계정에 알레르기 정보를 영구 저장하지 않는다.
- 품절 대체는 같은 매장의 동일 주 메뉴 카테고리, 보조 카테고리 일치 수, 원 가격 ±20%를 우선하고 중앙 판매·수량·영업·제공 시간을 재검증한다. 같은 매장에 후보가 없으면 원 매장 반경 3km의 다른 매장에 동일 주 카테고리·가격대·영업시간·방문 시각·인원 예약 가능 여부를 적용한다.
- 대체는 자동 적용하지 않는다. 같은 매장 대체는 사용자 수락 뒤 새 수량 선점과 원 수량 해제를 원자·보상 경계로 처리하고, 다른 매장은 새 예약 화면으로만 연결하며 기존 예약을 자동 취소하지 않는다.

## ADS-006 빈도 제한과 사용자 숨김

> 정책 상태: 확정

### 확정 규칙·권한·상태·흐름

- 광고는 `ADS-001`과 `ADS-003`~`ADS-005`의 향후 결정으로 유효한 상품·소재·지면·타게팅 입력이 존재할 때만 노출한다. 이 정책은 상품, 지면, 과금, 소재 적법성, 타게팅 항목이나 제공자를 정하지 않으며 입력이 없으면 광고를 표시하지 않는다.
- 사용자 구독 후보의 광고 제거는 광고 상품과 사용자 구독 플랜이 실제 출시되고 유효한 권한 버전이 명시될 때만 적용할 수 있다. 현재는 광고 상품과 사용자 구독이 모두 미출시이므로 제거할 광고나 활성 광고 제거 권한이 없으며, 광고 부재를 구독 혜택으로 표시하지 않는다.
- 활성 광고마다 캠페인·소재·지면별 버전된 빈도 정책을 서버가 적용한다. 정책이 누락되거나 카운터를 확인할 수 없으면 해당 광고를 실패 폐쇄하고 광고가 아닌 일반 콘텐츠 공간으로 처리하며 임의 기본 횟수를 발명하지 않는다.
- 사용자는 노출된 광고를 숨길 수 있고 로그인 사용자의 숨김은 중앙 계정 설정에, 비회원의 숨김은 추가 식별정보 없는 짧은 수명의 현재 브라우저 설정에 둔다. 숨김은 일반 추천·매장 검색 순위를 낮추거나 광고 외 콘텐츠를 제거하지 않는다.
- 노출 후보는 `허용`, `빈도 초과`, `사용자 숨김`, `정책/안전 차단`으로 판정하고 실제 표시가 확인된 노출만 중앙 원장에 한 번 기록한다. 클릭·구매 추정이나 알림 성공을 노출로 대신 세지 않는다.

### 동시성·실패·복구·알림·감사·개인정보

- 사용자 또는 비식별 브라우저 범위-캠페인-지면-정책 기간의 중앙 카운터와 노출 멱등 키로 여러 인스턴스의 중복 표시를 막는다. 카운터 갱신과 노출 허용을 원자 처리하고 응답 재전송은 같은 판정을 반환한다.
- 노출 기록 지연·중복 사건은 원장 ID로 대사하고, 숨김 해제·캠페인 종료·정정은 새 버전부터 적용한다. 이미 표시된 광고를 소급 삭제하거나 새 과금 책임을 만들지 않는다.
- 숨김 결과를 즉시 화면에 반영하고 설정이 계정 전체인지 현재 브라우저인지 설명한다. 정책 버전·허용/차단 사유·노출 원장·숨김 변경을 감사하되 생년월일·주류 이용 가능일, 정밀 위치, 기기 지문이나 외부 프로필을 수집·전달하지 않는다.

### 관측 가능한 인수 조건

- 광고 상품·정책·카운터가 없거나 장애이면 광고가 노출되지 않고 일반 추천 결과에는 영향이 없다.
- 병렬 요청·새로고침·다중 인스턴스에서도 같은 노출이 중복 계수되지 않고 버전 정책의 빈도 한도를 넘지 않는다.
- 사용자 숨김 뒤 해당 범위 광고는 차단되며 외부 사업자에게 연령 원본·추가 식별정보가 전달되지 않는다.

## ADS-007 추천 랭킹 요인과 가용성

> 정책 상태: 확정

### 확정 일반 추천·데이터·흐름

- 일반 추천은 `ADS-002`에 따라 광고 후보와 별도 파이프라인·결과 영역으로 계산하며 광고비·광고 계약·구독 여부를 일반 순위 가중치로 사용하지 않는다. 광고 후보가 일반 추천에 섞이거나 일반 추천으로 표시되는 전이는 금지한다.
- 후보는 공개 가능한 활성 매장·메뉴·행사 중 요청 시점의 검색 조건, 영업/판매 상태, 예약·수량·회차의 최신 가용 여부를 통과한 항목으로 제한한다. 순위는 사용자가 명시한 필터·취향, 거리 등 현재 요청 맥락, 집계된 품질·인기와 최신성 같은 버전된 요인을 사용하되 민감정보나 비공개 결제 규모를 입력하지 않는다.
- 후보 생성 시점과 화면 표시 직전에 중앙 가용 상태를 재검사한다. 추천은 발견 가능성일 뿐 자원 확보 보장이 아니며 예약·홀드 명령은 각 원 도메인의 최종 원자 검증을 다시 거친다.
- 랭킹 모델·요인·동점 규칙·후보 필터는 버전 관리하고 같은 입력 스냅샷과 버전에는 결정적 결과를 낸다. 개인 입력이 없으면 공개 카탈로그와 비식별 집계 요인만 쓰는 일반 순위로 전환한다.

### 동시성·실패·재처리·투명성·개인정보

- 검색 인덱스·가용 이벤트는 중복 제거와 버전 조건으로 반영하고, 늦은 이벤트가 최신 영업·재고 상태를 되돌리지 못하게 한다. 원본은 중앙 도메인 상태이며 인덱스·캐시 불일치는 재색인/무효화로 복구한다.
- 랭킹 서비스나 인덱스 장애에는 오래된 개인화 결과를 성공으로 가장하지 않고 최신 중앙 가용성을 확인할 수 있는 일반 검색으로 축소하거나 결과를 보류한다. 재처리는 모델 버전과 입력 체크포인트로 멱등 수행한다.
- 사용자에게 광고 아님과 주요 추천 요인, 가용성 확인 시각을 표시한다. 후보 제외·모델 버전·가용성 불일치·축소 동작을 감사·관측하되 새 PII, 지속적 기기 지문이나 외부 데이터 구매를 요구하지 않는다.

### 관측 가능한 인수 조건

- 광고비·구독 상태를 바꿔도 동일한 일반 추천 입력의 순위가 달라지지 않고 광고 후보가 일반 추천 표시에 섞이지 않는다.
- 품절·휴점·종료 상태는 표시 전 재검사에서 제외되고, 추천을 통과해도 최종 예약 자원은 원 도메인 검증 없이 확보되지 않는다.
- 중복·역순 가용 이벤트, 인덱스 재구축과 혼합 버전 배포 후에도 같은 입력·버전은 같은 순위로 수렴한다.

## ADS-008 추천 설명·편향·콜드스타트

> 정책 상태: 확정

### 확정 설명·콜드스타트·편향 통제

- 각 일반 추천에는 적용된 주요 요인 중 사용자가 이해할 수 있는 이유를 표시한다. 정확한 내부 가중치나 악용 가능한 임계치를 공개하지 않되, 광고·협찬이면 일반 설명을 사용하지 않고 `ADS-002`의 광고 표시를 우선한다.
- 개인 이용 이력이 없거나 부족하면 공개 카탈로그의 현재 가용성, 사용자가 명시한 조건과 개인정보를 최소화한 집계 인기·품질을 사용하는 비개인화 콜드스타트로 전환한다. 다른 사용자의 프로필을 유사하다고 추정하거나 새 민감정보를 수집하지 않는다.
- 후보 자격과 요인 정의는 모든 매장에 동일한 버전으로 적용하고 신규·소규모 매장을 결측값만으로 영구 제외하지 않는다. 노출·클릭·예약 전환의 분포와 제외 사유를 비식별 집계로 관측하되 특정 노출 비율이나 사업적 우대를 이 정책에서 보장하지 않는다.
- 사용자가 명시적 취향·필터를 수정·삭제하면 이후 추천에서 제거하고 관련 캐시를 무효화한다. 설명은 실제 사용된 요인에서 생성하며 템플릿 추측으로 존재하지 않은 이유를 표시하지 않는다.

### 동시성·실패·재평가·감사·개인정보

- 랭킹 결과 ID-모델 버전별 유일성과 설명 생성 멱등 키로 설명을 함께 고정하여 여러 인스턴스에서 결과와 설명이 어긋나지 않게 한다. 설명 생성 실패에는 설명 없는 개인화 추천을 표시하지 않고 비개인화 결과로 축소하거나 결과를 보류한다.
- 모델·요인 변경 시 고정 평가 데이터와 비식별 집계로 신규/기존 매장, 지역·카테고리별 제외와 성능 변화를 재평가한다. 이상은 배포 중지·직전 안전 버전 복귀 대상으로 기록하고 기존 사용자 행동을 소급 재해석하지 않는다.
- 모델 버전·주요 요인·콜드스타트 전환·설명 실패·집계 편향 신호를 감사·관측한다. 취향 원문을 광고 타게팅으로 전용하거나 생년월일·연락처·정밀 위치·민감속성을 새로 수집하지 않는다.

### 관측 가능한 인수 조건

- 모든 표시된 일반 추천의 설명이 실제 결과 ID·모델 버전의 사용 요인과 일치하고 광고는 일반 추천 설명으로 위장되지 않는다.
- 이력 없는 사용자에게 추가 개인정보 요구 없이 비개인화 결과가 생성되며 결측값만으로 신규 매장이 영구 제외되지 않는다.
- 설명 장애·모델 롤백·취향 삭제 뒤 캐시와 결과가 안전 버전·현재 입력으로 수렴하고 편향 관측은 비식별 집계만 사용한다.
