# 리뷰·신뢰·어뷰징 정책

> 문서 상태: 세부 정책 확정
> 정책 범위: `TRUST-001`~`TRUST-012`
> 최종 변경일: 2026-07-22

방문 인증 리뷰, 콘텐츠 기준, 신고, 식당 답글, 조작 탐지, 신뢰 점수, 제재 및 오탐 복구를 다룬다.

## 분류 현황

| 상태 | 중요도 | 정책 ID |
|---|---|---|
| 확정 | 필수 | `TRUST-001`, `TRUST-002`, `TRUST-004`~`TRUST-009`, `TRUST-011`, `TRUST-012` |
| 확정 | 권장 | `TRUST-003`, `TRUST-010` |

## 리뷰·검수 공통 불변식

- 리뷰 자격의 원본은 예약·홀드·테이스팅·체크인 도메인이 중앙 상태로 확정한 실제 완료 증거다. 결제 시도, 예약 생성, 매장 근처 위치, 알림 수신이나 사용자 주장만으로 자격을 만들지 않는다.
- 공개 콘텐츠와 검수 상태는 중앙 저장소에서 버전 관리한다. 자동 규칙은 위험을 표시하거나 제한된 임시 숨김만 수행하며 최종 제재·영구 삭제를 단독 확정하지 않는다.
- 자동·수동 검수는 MiriYum 내부 규칙과 권한 있는 운영자로 수행한다. 유료 외부 검수·프로파일링 서비스, 지속적 기기 지문, 정밀 위치나 새로운 신원정보를 기본 전제로 두지 않는다.
- 신고·검수·제재·복구는 근거 버전, 행위자, 상태 전후와 이의 결과를 감사하고, 공개 화면이나 식당에는 신고자 신원·비공개 증빙을 노출하지 않는다.

## TRUST-001 리뷰 작성 자격

> 정책 상태: 확정

### 확정 규칙·사전 조건·흐름

- 로그인한 일반 사용자가 자신의 거래에 대해 서버가 `이행 완료`로 확정한 증거를 가진 경우에만 리뷰 초안을 만들고 게시한다. 유효 증거는 방문 체크인 후 완료, 수령 완료 또는 행사 참가 완료처럼 원 도메인이 거래 ID와 사용자 ID에 연결해 확정한 사건으로 한정한다.
- 취소·노쇼·대기·미결제·결과 불명확 상태, 매장 위치 추정, 영수증 사진이나 운영자 메모만으로 자격을 인정하지 않는다. 완료 증거가 보정 취소되면 새 게시를 막고 기존 리뷰는 `자격 재검토`로 전환하되 즉시 영구 삭제하지 않는다.
- 완료 거래-작성자별 활성 리뷰는 하나이며 작성, 수정, 삭제는 해당 사용자만 수행한다. 운영자는 증거 없이 리뷰를 대신 작성하거나 자격을 부여할 수 없다.

### 동시성·실패·알림·감사·개인정보

- 완료 증거 ID, 리뷰 유일성 제약, 멱등 키와 버전 조건부 갱신으로 병렬 게시·재전송에도 하나만 생성한다. 원 도메인 장애나 증거 버전 불일치에는 게시를 실패 폐쇄하고 초안만 유지한다.
- 늦거나 역순인 완료·취소 사건은 중앙 거래 버전으로 재평가하고, 자격 변경은 검수 사건 하나로 멱등 처리한다. 사용자에게 게시·보류·자격 재검토와 이의 경로를 알린다.
- 리뷰에는 완료 증거의 내부 참조와 판정 버전만 저장하고 결제수단, 동행자, 위치 원본이나 현장 신분증을 복제하지 않는다. 자격 판정·게시·보류·복구를 감사한다.

### 관측 가능한 인수 조건

- 완료 증거 없는 취소·노쇼·대기 거래로는 리뷰가 게시되지 않는다.
- 같은 완료 거래에 병렬 게시해도 활성 리뷰 하나만 존재하고 다른 사용자의 거래로 작성할 수 없다.
- 완료 증거 정정·이벤트 역순 뒤에도 자격 상태가 중앙 거래와 수렴하고 자동 영구 삭제 없이 이의가 가능하다.

## TRUST-002 매장·메뉴 평가 구조

> 정책 상태: 확정

### 확정 규칙·데이터·흐름

- 리뷰는 `TRUST-001` 완료 증거가 가리키는 매장 평가 하나를 기본 단위로 삼고, 원 거래에 포함됐거나 서버가 완료와 연결한 메뉴에만 선택적 메뉴 평가를 붙인다. 다른 매장·미이용 메뉴를 임의 추가하지 못한다.
- 매장 평가와 메뉴별 평가는 서로 다른 대상 ID·버전으로 저장하고 메뉴 이름 변경·숨김·병합 뒤에도 당시 대상 스냅샷과 현재 대상 연결을 보존한다. 매장 점수로 메뉴 점수를 채우거나 반대로 추정하지 않는다.
- 공개 집계는 게시 가능 상태인 평가만 대상별로 분리 계산하며 수정·삭제·복구 사건을 재집계 입력으로 발행한다. 표본·표시 기준은 분석 집계와 분리된 버전 정책으로 동일 적용한다.

### 동시성·실패·감사·개인정보

- 완료 증거-대상 유일성, 리뷰 버전과 조건부 갱신으로 중복 대상 평가를 막는다. 메뉴 연결 실패는 매장 평가를 되돌리지 않고 해당 메뉴 평가만 보류해 재처리한다.
- 메뉴 카탈로그 장애에는 새 메뉴 평가를 실패 폐쇄하며 원 거래 스냅샷으로 존재하지 않은 메뉴를 추정하지 않는다. 집계 지연은 마지막 완료 시각과 지연 상태를 표시하고 원본 리뷰를 변경하지 않는다.
- 작성자에게 평가 대상과 공개 상태를 알리고 대상 추가·변경·집계 사건을 감사한다. 주문 메모·동행자나 결제 상세는 평가 대상 판정에 복제하지 않는다.

### 관측 가능한 인수 조건

- 한 완료 거래로 다른 매장이나 이용 근거 없는 메뉴를 평가할 수 없다.
- 메뉴 변경·삭제와 병렬 수정 후에도 매장/메뉴 평가가 섞이지 않고 당시 대상 참조를 재현할 수 있다.
- 집계 실패·재시작 뒤 대상별 공개 집계가 현재 게시 가능 버전과 일치한다.

## TRUST-003 별점과 취향 태그

> 정책 상태: 확정

### 확정 규칙·검증·흐름

- 별점은 대상별 1~5의 정수 하나로 받고 빈 값·범위 밖·소수 입력은 거부한다. 취향 태그는 선택 사항이며 중앙에서 승인한 버전된 목록에서 중복 없이 선택한다.
- 태그는 음식·경험 취향 표현에 한정하고 건강, 종교, 정치, 인종 등 민감정보 추론용 자유 입력이나 숨은 타게팅 분류로 사용하지 않는다. 태그 목록 변경은 기존 리뷰의 당시 태그 버전을 보존하고 새 작성·수정부터 적용한다.
- 별점과 태그는 매장·메뉴 대상에 독립 저장하고 공개 집계에는 현재 게시 가능 리뷰의 확정 버전만 반영한다.

### 동시성·실패·투명성·감사

- 리뷰 버전과 멱등 키로 중복 수정·역순 이벤트를 막고 집계 소비자는 리뷰 ID-버전별 한 번만 반영한다. 태그 정책을 확인할 수 없으면 태그 저장만 거부하고 유효한 별점 초안은 보존한다.
- 사용자에게 사용한 척도와 태그 의미를 표시하고 수정·집계 반영 지연을 알린다. 입력·정책 버전·정규화·거부·집계 사건을 감사하되 취향 태그를 개인 제재나 광고 타게팅으로 자동 전용하지 않는다.

### 관측 가능한 인수 조건

- 0·6·소수 별점과 승인 목록 밖 태그가 저장되지 않는다.
- 태그 목록 배포 중 여러 인스턴스가 같은 버전을 적용하고 기존 태그가 소급 변환되지 않는다.
- 중복 수정·집계 재처리에도 리뷰당 최신 별점·태그가 정확히 한 번 집계된다.

## TRUST-004 사진·텍스트 콘텐츠 기준

> 정책 상태: 확정

### 확정 기준·상태·처리

- 실제 이용 경험과 관련된 사진·텍스트만 허용한다. 불법물, 위협·혐오·괴롭힘, 노골적 성적 콘텐츠, 타인의 연락처·신분증·얼굴 등 사생활 침해, 사칭, 악성 코드·외부 광고와 무관한 홍보는 게시하지 않거나 신고 후 제한한다.
- 입력은 `초안`, `처리 중`, `게시 가능`, `검수 보류`, `숨김`, `거부`, `복구`로 관리한다. 이미지는 허용 형식·크기·실제 디코딩을 검증하고 메타데이터를 제거한 처리본만 공개하며 원본 처리 실패 시 텍스트까지 성공으로 가장하지 않는다.
- 명백한 실행파일·손상 파일·비공개정보 패턴은 자동 차단할 수 있고 맥락 판단이 필요한 콘텐츠는 내부 운영자 검수로 보낸다. 유료 외부 검수 API를 필수 통제로 사용하지 않는다.

### 동시성·실패·통지·감사·개인정보

- 업로드 ID·내용 무결성 값·리뷰 버전으로 재처리를 멱등화하고, 공개 상태와 처리된 파일 참조를 일관되게 전환한다. 작업 실패는 비공개 상태로 제한 재시도하고 소진 시 검수 큐와 경보로 보낸다.
- 작성자에게 거부·숨김의 기준 범주와 이의 경로를 알리되 악용 가능한 탐지 세부나 신고자 정보는 제공하지 않는다. 규칙 버전·파일 무결성 값·자동 판정·운영자 결정·복구를 감사한다.
- 로그·이벤트에 원문·원본 URL·EXIF를 남기지 않고 검수자는 배정 사건의 최소 자료만 짧은 접근 경로로 조회한다. 보관·삭제는 PRIV 정책을 따른다.

### 관측 가능한 인수 조건

- 실행파일·손상 이미지·메타데이터 포함 원본은 공개되지 않고 처리된 허용 파일만 노출된다.
- 처리 작업 중복·재시작에도 공개 파일 버전 하나만 연결되고 실패 파일이 임시 공개되지 않는다.
- 숨김·거부마다 근거와 이의 경로가 있으며 운영자 복구로 같은 콘텐츠 버전을 감사 가능하게 되돌릴 수 있다.

## TRUST-005 수정·삭제·보존

> 정책 상태: 확정

### 확정 규칙·상태·재처리

- 작성자는 자신의 리뷰를 수정하거나 공개 삭제할 수 있다. 수정은 새 불변 버전을 만들고 최신 승인 버전만 공개하며, 삭제는 공개 본문·사진·별점·태그를 즉시 제외하는 `사용자 삭제` 상태로 전환한다.
- 운영자는 감사 근거 없이 작성자 내용을 고쳐 쓰지 않는다. 정책 위반은 별도 숨김·거부 상태로 처리하고, 오판 복구 시 마지막 유효 작성자 버전을 복원한다.
- 삭제·수정 사건은 매장·메뉴 집계, 검색, 캐시와 분석 파생물에 보정 이벤트로 전파한다. 법정 보존이나 보관 기간은 PRIV의 TODO 결정을 따르며, 그 전까지 일반 서비스에는 내용 대신 최소 사건·무결성 참조만 남긴다.

### 동시성·실패·알림·감사·개인정보

- 리뷰 버전 조건부 갱신으로 수정·삭제·검수 경합 중 유효한 전이 하나만 성공시킨다. 중복 삭제는 같은 결과를 반환하고 늦은 수정은 삭제 상태를 되돌리지 않는다.
- 파생 시스템 실패는 원본 공개 삭제를 지연시키지 않고 재처리 큐에서 누락된 삭제·보정만 멱등 재전파한다. 사용자에게 수정·삭제·복구와 파생 반영 지연을 알린다.
- 작성자·운영자·사유·버전·파생 전파 결과를 감사하되 삭제된 원문을 일반 로그나 분석 이벤트에 복제하지 않는다.

### 관측 가능한 인수 조건

- 삭제 직후 공개·검색·신규 집계에서 제외되고 늦은 수정 이벤트로 재등장하지 않는다.
- 수정·삭제·검수 병렬 처리에도 공개 최신 버전은 하나이며 모든 파생 집계가 보정 후 수렴한다.
- 복구는 이력 없는 덮어쓰기가 아니라 근거·운영자·복원 버전이 남는 가역 전이로 수행된다.

## TRUST-006 신고·숨김·검수

> 정책 상태: 확정

### 확정 규칙·권한·상태·흐름

- 로그인한 사용자와 자기 매장의 식당 대표자는 공개 리뷰·사진·답글을 사유 코드와 함께 신고할 수 있다. 운영자는 신고자 신원을 피신고자에게 공개하지 않고, 동일 신고자-대상-사유의 활성 신고는 하나로 합친다.
- 상태는 `접수`, `분류`, `임시 숨김`, `기각`, `위반 확정`, `복구`, `이의 중`으로 관리한다. 즉각적 안전 위험·명백한 개인정보 노출만 중앙 규칙으로 임시 숨김할 수 있고, 일반 불만이나 신고 수만으로 자동 위반 확정하지 않는다.

### 동시성·실패·통지·감사·개인정보

- 대상-검수 사건 유일성, 신고 멱등 키와 상태 버전으로 중복 접수·상충 결정을 막는다. 검수 작업 실패는 공개 위험도에 따라 기존 상태를 유지하고 재배정하며 자동 유죄로 처리하지 않는다.
- 작성자에게 임시 조치·최종 결과·사유 범주·이의 경로를 알리고 신고자에게 접수·종결만 알린다. 모든 신고를 악용한 괴롭힘은 별도 속도 제한·검토 대상으로 삼되 정상 신고를 자동 제재하지 않는다.
- 신고·자동 규칙·배정·결정·증거 버전·복구를 변경 불가능한 감사 이력에 남기고 신고자 신원과 비공개 증거는 최소 권한으로 분리한다.

### 관측 가능한 인수 조건

## TRUST-007 식당 답글과 분쟁

> 정책 상태: 확정

## TRUST-010 사용자 신뢰 점수

### 관측 가능한 인수 조건

- 같은 사건 집합과 산식 버전은 인스턴스·재처리 횟수와 무관하게 같은 결과를 낸다.

## TRUST-011 제재 단계와 이의 제기

> 정책 상태: 확정

### 확정 단계·권한·상태 전이

- 리뷰 도메인 조치는 안내, 콘텐츠 숨김, 일정 기간 리뷰 작성 제한 순으로 비례 적용하고 심각한 안전 위협에는 앞 단계를 생략할 수 있다. 계정 전체 정지·영구 정지는 이 정책이 독자 확정하지 않고 AUTH-011의 권한·이중 승인·이의 절차를 따른다.

### 동시성·실패·통지·감사·개인정보

- 위반 사건별 활성 제재·이의 사건 유일성과 상태 버전으로 병렬 승인·취소 중 최초 유효 전이 하나만 반영한다. 기간 만료 작업은 중앙 시각과 멱등 획득으로 중복 해제를 막는다.
- 통지 실패는 안전 조치를 되돌리지 않고 제한 재시도·경보하되 이의 접수 경로는 앱에서 조회 가능하게 한다. 조치·근거·권한·통지·이의·복구를 감사하고 신고자·다른 사용자의 자료는 최소 공개한다.

### 관측 가능한 인수 조건

- 자동 탐지 하나만으로 영구 정지되지 않고 계정 제재는 AUTH-011 권한 경계를 우회하지 않는다.
- 동시 제재·만료·이의 결정에도 유효 상태 하나만 남고 취소된 조치의 공개·신뢰 파생물이 복구된다.
- 모든 제한에 사유 범주·범위·종료 조건·이의 경로와 감사 이력이 존재한다.

## TRUST-012 자동 탐지의 오탐 처리

> 정책 상태: 확정

### 확정 오탐·복구·재처리 흐름

- 자동 탐지로 제한된 작성자는 앱에서 대상, 근거 범주와 이의 경로를 확인하고 오탐 검토를 요청할 수 있다. 검토자는 원래 규칙 결과, 해당 콘텐츠 버전과 최소 완료 증거를 확인하며 신고자 신원·무관한 계정 활동을 요구하지 않는다.
- 오탐 확정 시 `오탐 복구` 사건으로 콘텐츠 공개 상태, 리뷰 작성 권한, 신뢰 점수와 검색·집계를 복원한다. 복구는 원 결정을 삭제하거나 덮어쓰지 않고 원 사건-보정 사건 관계와 운영자·사유를 보존한다.
- 탐지 규칙 수정에는 영향받은 보류·제재 사건을 규칙 버전별로 재처리한다. 재처리는 확정된 무혐의·복구를 자동 뒤집지 않고 새로운 독립 근거가 있을 때만 새 검수 사건을 만든다.

### 동시성·실패·통지·감사·개인정보

- 이의 사건 유일성, 복구 멱등 키와 파생 시스템별 처리 체크포인트로 중복 복원·중복 집계 보정을 막는다. 일부 복구 실패는 성공한 복구를 되돌리지 않고 남은 파생물만 재처리한다.
- 사용자에게 접수·복구·유지 결과와 파생 반영 상태를 알리고, 반복 오탐률·복구 지연·미완료 파생 보정을 관측·경보한다. 원/보정 사건·규칙 버전·검토자·재처리 결과를 감사한다.
- 오탐 분석은 기존 사건의 최소 참조와 비식별 집계만 사용하며 외부 유료 검수·추가 PII 수집으로 확대하지 않는다.

### 관측 가능한 인수 조건

- 오탐 확정 뒤 콘텐츠, 작성 권한, 신뢰 점수와 공개 집계가 보정 사건을 통해 원래 유효 상태로 수렴한다.
- 복구 작업 중단·재실행에도 같은 파생물이 중복 더해지거나 콘텐츠가 여러 버전으로 공개되지 않는다.
- 규칙 재처리가 확정 무혐의를 근거 없이 뒤집지 않고 모든 유지·복구 결과와 사용자 통지가 추적된다.
