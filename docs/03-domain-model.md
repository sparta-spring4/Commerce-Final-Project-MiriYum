# 초기 도메인 모델

## 목적과 경계

초기 구현은 아래 다섯 도메인만 패키지·책임 경계로 사용한다. 상세 규칙의 단일 원본은 [서비스 정책 마스터](service-policies/README.md)와 연결된 정책 문서이며, 이 문서는 데이터 모델이나 저장 구조를 확정하지 않는다.

예약, 메뉴 홀드, 웨이팅, 체크인은 초기에는 모두 `booking`이 소유한다. 이 책임은 변경 주기와 복잡성이 충분히 커진 뒤 ADR로 분리 여부를 결정한다.

## `auth`

- **책임:** 회원 식별, 로그인 세션, 계정 수명주기와 접근 주체 확인을 일관되게 제공한다.
- **소유 상태:** 계정 상태, 인증 수단 연결 상태, 세션·기기 상태, 기본 프로필 및 동의 상태.
- **제공 기능:** 가입·로그인·로그아웃, 계정 복구·잠금, 역할 확인, 탈퇴 및 계정 보호 절차.
- **허용 의존 대상:** `notification`(보안·계정 안내), `store`(대표자 권한 확인), `booking`·`payment`(인증된 주체 확인).
- **정책 연결:** [회원·인증·계정](service-policies/01-member-auth.md), [개인정보·보안](service-policies/17-privacy-security.md), [운영자·분쟁·수동 복구](service-policies/15-admin-operation.md).

## `store`

- **책임:** 식당의 입점, 대표자 권한, 영업 가능 여부와 고객에게 보이는 매장·메뉴 정보를 관리한다.
- **소유 상태:** 입점 심사·기능 활성화 상태, 매장 운영 상태, 영업 시간, 메뉴와 판매 가능 상태.
- **제공 기능:** 입점 신청과 보완 처리, 대표자 운영 권한, 매장·메뉴 등록 및 변경, 영업·휴점 전환, 식당 탐색에 필요한 공개 정보 제공.
- **허용 의존 대상:** `auth`(대표자·운영 주체 확인), `booking`(운영 가능 시간·수용량 참조), `notification`(입점·운영 안내).
- **정책 연결:** [매장 입점·심사·대표자 권한](service-policies/02-store-onboarding.md), [매장 운영·영업시간·메뉴](service-policies/03-store-operation.md), [광고·추천](service-policies/13-ad-recommendation.md), [분석·수요 리포트](service-policies/14-analytics-report.md).

## `booking`

- **책임:** 방문 기회와 현장 방문 순서를 공정하고 일관되게 조정한다. 초기에는 예약, 메뉴 홀드, 웨이팅, 체크인·노쇼, 취소 자리·수량 승계를 함께 소유한다.
- **소유 상태:** 예약·수용량·임시 선점 상태, 메뉴 홀드 수량과 만료 상태, 웨이팅 순번과 호출 상태, 체크인·노쇼 판정 상태, 승계 제안 상태.
- **제공 기능:** 예약 생성·변경·취소, 수용량 확인, 메뉴 홀드, 웨이팅 등록·호출, 체크인·노쇼 처리, 취소분 승계 제안.
- **허용 의존 대상:** `auth`(방문자·대표자 식별), `store`(영업·메뉴·수용량 기준), `payment`(결제 결과 확인), `notification`(예약·웨이팅 사건 전달).
- **정책 연결:** [방문 예약](service-policies/04-reservation.md), [현장·원격 웨이팅](service-policies/05-waiting.md), [메뉴 홀드](service-policies/06-menu-hold.md), [체크인·노쇼](service-policies/09-checkin-noshow.md), [취소 자리·수량 자동 승계](service-policies/10-waitlist-transfer.md), [대규모 트래픽·분산 환경·장애 복구](service-policies/18-scale-reliability.md).

## `payment`

- **책임:** 금액이 발생하는 예약 관련 결제와 취소·환불 결과를 신뢰할 수 있게 조정한다.
- **소유 상태:** 결제 요청·승인·실패 상태, 환불 상태, 결제 수단 처리 결과, 정산·대사 상태.
- **제공 기능:** 결제 요청의 중복 방지, 결제 결과 수신, 전체·부분 환불, 예약과 결제 상태의 결속, 대사와 운영 복구 지원.
- **허용 의존 대상:** `auth`(결제 주체 확인), `booking`(결제 대상·취소 사건), `store`(식당 귀속 정보), `notification`(결제·환불 결과 안내).
- **정책 연결:** [결제·환불·정산](service-policies/08-payment-refund.md), [사용자·식당 구독](service-policies/11-subscription.md), [운영자·분쟁·수동 복구](service-policies/15-admin-operation.md).

## `notification`

- **책임:** 다른 도메인의 확정 사건을 수신 동의와 전달 보장 기준에 맞춰 사용자에게 전달한다.
- **소유 상태:** 수신 동의와 채널 선호, 발송 요청·전달·실패·재시도 상태, 알림 이력.
- **제공 기능:** 사건 기반 알림 요청 접수, 채널 선택, 중복 방지, 실패 재시도·대체 채널, 수신 이력 제공.
- **허용 의존 대상:** `auth`(수신자와 동의 확인), `store`·`booking`·`payment`(발송 근거 사건 수신). 다른 도메인의 업무 상태를 변경하지 않는다.
- **정책 연결:** [알림](service-policies/16-notification.md), [개인정보·보안](service-policies/17-privacy-security.md), [대규모 트래픽·분산 환경·장애 복구](service-policies/18-scale-reliability.md).

## 경계 원칙

- 각 도메인은 자신의 상태 전이와 정책 해석을 소유하고, 다른 도메인은 제공 기능을 통해서만 필요한 사실을 얻는다.
- 공통 보안·개인정보·운영·신뢰성 요구는 별도 초기 업무 도메인을 만들지 않고, 관련 도메인의 정책 연결과 구현 기준으로 적용한다.
- 코스·테이스팅, 구독, 리뷰·신뢰·어뷰징, 광고 상품과 Pro·고급 분석 리포트는 정책 색인에 보존하되 초기 도메인 추가나 구현 약속으로 해석하지 않는다. 무료 기본 추천과 Free 기본 운영 통계는 초기 `store` 책임에 포함하며 세부 경계는 각 정책 원본을 따른다.
